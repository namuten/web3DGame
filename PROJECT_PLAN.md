# 🚀 프로젝트 개발 로드맵 (Three.js 멀티플레이어 아키텍처)

이 계획은 `CLAUDE.me`의 구조와 개발 로드맵 지침을 기반으로, 프로젝트를 성공적으로 구축하기 위한 6단계 마일스톤으로 세분화된 계획입니다.

---

### 단계 1: 프로젝트 초기화 및 기본 환경 구성 (Project Setup) [완료]
*클라이언트와 서버 등 전체 프로젝트의 골격을 잡는 단계입니다.*

1. ✅ **저장소 구성**: 클라이언트(`client/`)와 웹소켓 서버(`server/`) 디렉토리 분리 및 모노레포 구조 세팅
2. ✅ **패키지 환경 설정**: 
   - Frontend: Vite, TypeScript, Three.js 설치 및 `vite.config.ts` 설정
   - Backend: Node.js, TypeScript, Express, Socket.IO 설치
3. ✅ **코드 구조화**: 
   - `client/src/` 하위에 `engine`, `game`, `network`, `ui` 폴더 생성
   - `server/` 하위에 기본 서버 구조 세팅
4. ✅ **린트 및 포맷 설정**: 기본적인 TypeScript 환경 및 핫-리로딩 설정 완료

### 단계 2: 싱글 플레이어 3D 환경 구축 (Client - Three.js Single Player Scene) [완료]
*네트워크 연동 전에 클라이언트가 브라우저에서 렌더링되고 동작하도록 만듭니다.*

1. ✅ **렌더링 엔진 기본 세팅**: `renderer.ts`, `scene.ts`, `camera.ts` 작성 및 브라우저 창 렌더링 확인
2. ✅ **플레이어 컨트롤 구현**: WASD 이동, 방향키 카메라 회전(오비탈), 스페이스 점프 로직 구현
3. ✅ **월드 환경 구성**: 사이버펑크 스타일 그리드 바닥, 랜덤 장애물 박스, 네온 조명 설정
4. ✅ **기본 UI 레이어 추가**: HUD(조작 가이드), 채팅 입력창 연동

### 단계 3: 웹소켓 서버 구성 및 기반 마련 (Realtime Server & Connection) [완료]
*유저들이 접속할 수 있는 실시간 서버의 통신 기반을 만듭니다.*

1. ✅ **네트워크 서버 연결**: 클라이언트에 `socket.ts` 구현, 서버에 Express + Socket.IO 세팅
2. ✅ **접속/종료 관리 로직**: 서버에서 플레이어 ID 관리 및 `player_joined`, `player_left` 이벤트 처리
3. ✅ **통신 프로토콜 설계**: `MOVE`, `STATE_UPDATE`, `CHAT_MESSAGE`, `SHOOT` 프로토콜 정의 및 구현

### 단계 4: 플레이어 동기화 (Game State Synchronization) [완료]
*본격적인 멀티플레이 경험을 제공하기 위한 위치 및 상태 동기화 단계입니다.*

1. ✅ **상태 동기화 모델 구축**: 클라이언트 위치 서버 전송 및 서버 브로드캐스트 시스템 완료
2. ✅ **클라이언트 타유저 렌더링**: 다른 플레이어 접속 시 빨간색 캐릭터로 렌더링 및 실시간 이동 동기화
3. ✅ **기능 확장**: 방향키 기반 카메라 회전에 따른 이동 방향(Camera-relative) 연동 완료

### 단계 5: 룸 스케일링 및 멀티플레이어 상태 확장 (Rooms & Multiplayer State) [진행중]
*단일 공간에서 벗어나 여러 방과 고급 상호작용을 처리합니다.*

1. ✅ **Room Manager 구현**: 기본 `DEFAULT_ROOM` 기반의 룸 격리 엔진 서버 측 구현
2. ✅ **채팅 시스템**: 한국어 IME(한글 조합) 버그 수정된 실시간 채팅 및 메시지 브로드캐스트 완료
3. ✅ **전투/상호작용 시스템**: 탄환 발사(F키/클릭), Raycast 충돌 판정, 명중 파티클, 1인 1발 제한 시스템 완료
4. ⏳ **로컬/글로벌 이벤트**: 아이템 획득 및 상호작용 가능한 NPC 추가 예정

### 단계 6: 영구 데이터 저장소 연동 및 배포 (Database, Auth, Infrastructure)
*데이터를 영구적으로 저장할 수 있는 데이터베이스와 시스템들을 통합합니다.*

1. **API 서버 구현**: REST API 엔드포인트를 통해 사용자 회원가입, 로그인 검증, 프로필 데이터 제공 역할 배정
2. **데이터베이스 (DB) 및 시스템 통합**: 
   - 유저 정보 및 전적 관련 스키마 구성 (MySQL / PostgreSQL / MongoDB 중 선택)
   - 세션 캐싱 및 Pub/Sub 통신용으로 Redis 추가하여 서버 간 이벤트 전송 구축
3. **호스팅 및 클라우드 배포**:
   - Vercel 등을 통한 Frontend 배포 
   - 서버 컨테이너화(Docker) 적용 및 Cloud Provider (AWS, GCP 등)에 배포
   - 데이터베이스 매니지드 서비스 연동
