현재 세션에서 작업한 내용을 `SESSION_SUMMARY.md` 파일 상단(첫 번째 `---` 구분선 바로 아래)에 추가하세요.

## 순서

1. `git diff HEAD` 와 `git log --oneline -10` 을 실행해 이번 세션의 변경 내용을 파악합니다.
2. 이번 대화에서 다룬 작업 내용도 함께 참고합니다.
3. 아래 형식으로 새 항목을 작성합니다.
4. `SESSION_SUMMARY.md` 파일의 첫 번째 `---` 구분선 **바로 아래**에 새 항목을 삽입합니다. 기존 내용은 절대 수정하지 않습니다.

## 작성 형식

```
## 📅 YYYY-MM-DD <세션 제목 (핵심 작업 1~2개 요약)>

<세션에서 달성한 내용을 2~3문장으로 요약>

### 🕹️ 주요 작업 내용

#### <이모지> 1. <작업 카테고리>
*   **<항목>**: <설명>

#### <이모지> 2. <작업 카테고리>
*   **<항목>**: <설명>

---
```

## 규칙

- 날짜는 오늘 날짜(YYYY-MM-DD)를 사용합니다.
- 제목은 이번 세션의 핵심 작업을 간결하게 표현합니다.
- 기술적인 내용은 구체적으로 (파일명, 함수명, 기능명 포함).
- 이모지를 적극 활용해 가독성을 높입니다.
- 한국어로 작성합니다.
- 삽입 후 변경된 파일 내용을 확인합니다.
